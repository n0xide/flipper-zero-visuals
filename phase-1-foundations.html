Phase 1 — Foundations

What the Flipper actually is, what's inside it, what gear you need around it, and which firmware to run. Treat this as the lab-bench setup chapter — once you can identify which chip handles each later phase, the rest of the curriculum makes sense as "we exercise this chip now."

Legality boundary on this page: the lightest of any chapter. The Flipper itself is legal to own and to take apart in every jurisdiction the curriculum targets. The hardware is yours — disassemble it, probe it, flash it. The constraints kick in later when you transmit, read, or inject into other people's systems. See Phase 0 for the policy framing and each later phase for domain-specific limits.

2. Which chip does what — curriculum phase by silicon

Every later phase exercises a specific subsystem. This is the map.

Subsystem / chipWhat it doesCurriculum phase(s)Limits
STM32WB55RG Main MCU. Dual-core (Cortex-M4 application + M0+ radio). 1 MB flash, 256 KB RAM. Runs FuriHAL + the application code. Handles USB, BLE, file system, all UI. foundation for everything; specifically BLE in Phase 6 BLE is host-only on stock (advertising + GATT central). No raw radio access without dev firmware.
CC1101 TI sub-GHz transceiver, SPI-attached to the WB55. 300–348 / 387–464 / 779–928 MHz; ASK/OOK + 2-FSK + 4-FSK + MSK + GFSK. Phase 2 sub-GHz ~10 mW TX. Limited by Flipper firmware to the regional ISM allowlist (unlock requires alternate firmware).
ST25R3916 NFC reader IC, SPI-attached. ISO 14443A/B + ISO 15693 + FeliCa + NFC-A/B/F. Drives the front-side NFC antenna coil. Phase 3 HF NFC Cannot fully emulate a card (passive ground-coupling); good for reads + simple tag emulation.
LF RFID frontend Discrete 125 kHz analog frontend on a separate antenna coil. Reads EM4100, HID Prox, Indala, FDX-B; emulates EM4100 + HID for cloning. Phase 3 LF RFID 125 kHz only; not all LF protocols supported in stock firmware.
1-Wire / iButton contact Steel back contact + 1-Wire bus driven by an STM32 GPIO. Reads + emulates DS1990A and several intercom variants (Cyfral, Metakom). Phase 4 iButton · also exposed on GPIO pin 17 Open-drain bus; works on owned hardware only by policy.
IR LED + TSOP receiver Front-mounted ~940 nm IR transmit LED and a TSOP-style 38 kHz bandpass receiver. Auto-classifies NEC / RC5 / SIRC; imports Pronto. Phase 4 IR One-way: TX yes, RX yes, but no protocol-level bidirectional channel. CEC over HDMI is a different beast.
USB-C (native USB) Native USB on the WB55. Used for charging, file transfer, and Bad USB HID emulation in attack mode. Phase 5 Bad USB Bad USB requires the Bad USB application to be loaded; otherwise just shows up as CDC + MTP.
18-pin GPIO header Side header exposing 13 user-I/O pins (muxed for UART, SPI, I²C, SWD, 1-Wire) plus +5V / +3.3V / 3× GND. The hardware-hacking surface. Phase 7 hardware 3.3 V logic; ~20 mA per pin; 5 V rail off by default.
microSD slot SPI-attached microSD. Stores the application file system, captures, NFC dumps, sub-GHz .sub files. required for most apps + capture work FAT32 only; the SD is essentially a requirement for any non-trivial use.
Buzzer Piezo buzzer for UI + Music Player + sub-GHz audio visualisation. UI + Music Player FAP Tone only; no PCM.

4. Firmware ecosystem — which one to run

Three live forks as of 2026. Pick by what you actually need; switching is non-destructive but each rewrite consumes a study session.

Official (OFW) — start here

Flipper Devices' official build. Conservative. The TX allowlist is enforced strictly per region; only well-vetted features ship.

  • Pros: stable; updates frequent; first-party support; legal-default behaviour.
  • Cons: no hardnested attack; no extended sub-GHz TX bands; smaller app catalogue.
  • Reach for it: always your first install; revert to it if anything misbehaves.

Momentum

Successor to the popular Xtreme firmware. Active development, large app catalogue, alternate UI elements, hardnested + extended NFC tooling.

  • Pros: hardnested; extended NFC kit; ships several Phase 6 helpers; large built-in FAP store.
  • Cons: drops region TX restrictions by default — this is your responsibility legally; minor stability quirks compared to OFW.
  • Reach for it: when Phase 3 hits hardened MIFARE Classic, or when you want the larger app library for Phase 4-7 helpers.

Xtreme (legacy)

The original "everything unlocked" community firmware. Development largely moved into Momentum in 2024–2025 but Xtreme builds still exist and some study guides reference it.

  • Pros: historical reference; some legacy plugins still target it; archive of older app versions.
  • Cons: slower release cadence than Momentum; some Xtreme-specific apps don't work on current OFW or Momentum.
  • Reach for it: only to follow a specific Xtreme-era guide. Otherwise prefer Momentum or OFW.
Switching firmware: backs up the internal storage; reflashing wipes the WB55's user partition (SD card is untouched). Save your .nfc / .sub / .ir captures before each flash. Roll back with the official qFlipper desktop tool if anything misbehaves.
Legality note on unlocked TX: Momentum and Xtreme remove the regional TX allowlist by default. This does not grant a license to transmit outside what your jurisdiction permits — it just removes the firmware's training wheels. FCC Part 15 (US) and CEPT (EU) still apply. The Flipper's TX power is low, but jamming, replay at out-of-scope targets, or operating in licensed bands is still illegal regardless of which firmware is running.
